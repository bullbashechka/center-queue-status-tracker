# 005. Добавление, редактирование, удаление и архивирование ребёнка

## Этап

Админка и управление очередью.

## Цель

Реализовать полный административный CRUD для ребёнка с валидацией обязательных полей, генерацией персональной ссылки и архивированием при удалении.

## Контекст из PRD

Сотрудник вручную заводит ребёнка. Обязательные поля: ФИО, ИИН 12 цифр, телефон родителя. Ориентировочный срок необязателен. При первом добавлении генерируется постоянная ссылка.

## Scope

- Форма добавления ребёнка.
- Форма редактирования ФИО, ИИН, телефона и ориентировочного срока.
- Валидация ИИН и телефона.
- Генерация уникальной персональной ссылки.
- Архивирование записи при удалении.
- Разрешение повторного добавления ИИН после архивирования старой записи.

## Out of Scope

- Родительская подача заявки.
- Полное физическое удаление данных из базы.
- Дополнительные персональные поля.

## Требования

- ФИО, ИИН и телефон обязательны.
- ИИН должен состоять из 12 цифр.
- ИИН должен быть уникальным среди активных записей.
- Удаление должно требовать подтверждения.
- Удалённая запись не показывается родителю через ссылку или поиск по ИИН.

## Acceptance Criteria

- Сотрудник может добавить ребёнка с валидными обязательными полями.
- При добавлении создаётся персональная ссылка.
- Невалидный ИИН или дублирующий активный ИИН блокирует сохранение.
- Сотрудник может изменить ФИО, ИИН, телефон и срок.
- После удаления запись исчезает из активного списка и недоступна родителю.
- После удаления можно создать новую активную запись с тем же ИИН.

## Зависимости

Задачи 002, 003 и 004.

## Implementation Notes

Удаление в этой задаче означает архивирование, а не физическое удаление.
