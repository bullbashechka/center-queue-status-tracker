# 008. Публичный поиск по ИИН и ограничение попыток

## Этап

Публичные страницы для родителей.

## Цель

Реализовать публичную страницу поиска статуса по ИИН ребёнка с ограничением попыток перебора.

## Контекст из PRD

Публичная ссылка ведёт на форму поиска. Родитель вводит ИИН ребёнка. Если запись найдена, открывается та же страница статуса, что и по персональной ссылке.

## Scope

- Публичная форма поиска по ИИН.
- Валидация формата ИИН.
- Поиск только среди доступных записей.
- Переход на страницу статуса при успешном поиске.
- Сообщение «заявка не найдена» без деталей.
- Ограничение 5 попыток за 15 минут с одного IP-адреса/устройства.

## Out of Scope

- Авторизация родителя.
- Раскрытие ФИО или статуса прямо в результатах поиска до открытия страницы статуса.

## Требования

- Поиск не требует входа.
- Недоступные/архивные записи не должны открываться через поиск.
- Ограничение попыток должно работать для успешных и неуспешных попыток.
- После превышения лимита поиск временно блокируется до окончания периода.

## Acceptance Criteria

- Родитель может ввести валидный ИИН и найти активную запись.
- При найденной записи открывается публичная страница статуса этого ребёнка.
- При неизвестном ИИН показывается нейтральное сообщение без подсказок.
- После 5 попыток за 15 минут дальнейшие попытки временно блокируются.
- Невалидный ИИН показывает ошибку формата на русском языке.

## Зависимости

Задача 007.

## Implementation Notes

Лимит должен снижать риск перебора, но не добавлять родительскую регистрацию, так как она вне MVP.
