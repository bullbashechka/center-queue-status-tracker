# 003. Авторизация сотрудников

## Статус

Выполнено.

## Этап

Доступ сотрудников.

## Цель

Закрыть админку от публичного доступа и реализовать вход сотрудников по логину и паролю.

## Контекст из PRD

Доступ в админку имеют только сотрудники центра. Все сотрудники имеют одинаковые права. Публичной регистрации нет.

## Scope

- Реализовать страницу входа.
- Реализовать сессию сотрудника.
- Защитить все маршруты админки.
- Подготовить создание учётных записей на этапе настройки системы.
- Реализовать выход из системы.

## Out of Scope

- Роли и уровни прав.
- Самостоятельная регистрация сотрудников.
- Восстановление пароля через email/SMS.

## Требования

- Любой авторизованный сотрудник может выполнять любые действия.
- Неавторизованный пользователь не должен видеть админку.
- Публичные страницы статуса и поиска не требуют входа.
- Пароли должны храниться в безопасном виде.

## Acceptance Criteria

- Сотрудник может войти с корректным логином и паролем.
- При неверных данных входа показывается русскоязычная ошибка.
- Админские страницы перенаправляют неавторизованного пользователя на вход.
- После выхода админские страницы снова недоступны без повторного входа.

## Зависимости

Задачи 001 и 002.

## Implementation Notes

Не добавлять роли: это явно вне MVP.
