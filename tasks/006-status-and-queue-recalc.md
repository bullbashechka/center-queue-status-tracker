# 006. Статусы, откат ошибок и пересчёт очереди

## Статус

Выполнено.

## Этап

Админка и управление очередью.

## Цель

Реализовать смену статусов ребёнка, подтверждённый откат ошибки и автоматический пересчёт активной FIFO-очереди.

## Контекст из PRD

Ребёнок проходит четыре этапа. Обычный сценарий — движение вперёд. При ошибке сотрудник может откатить статус назад после подтверждения. Статус «Зачислен» выводит ребёнка из активной очереди.

## Scope

- UI смены статуса.
- Проверка допустимых переходов.
- Подтверждение отката на более ранний статус.
- Исключение зачисленных детей из активной очереди.
- Пересчёт номеров после зачисления или архивирования.
- Подготовка событий для текстов уведомлений о смене этапа и продвижении очереди.

## Out of Scope

- Ручная перестановка детей в очереди.
- Автоматический расчёт срока начала занятий.

## Требования

- Статусы: «Документы приняты», «Диагностика пройдена», «Ожидание зачисления», «Зачислен».
- Статус виден сотруднику и родителю.
- После зачисления ребёнок не имеет номера в активной очереди.
- Пересчёт должен обновлять позиции всех затронутых активных детей.

## Acceptance Criteria

- Сотрудник может перевести ребёнка на следующий этап.
- При откате назад интерфейс требует подтверждение.
- После зачисления ребёнок пропадает из активной очереди.
- У детей, стоявших ниже, номер уменьшается на 1.
- Событие изменения статуса создаётся для подготовки текста сообщения.
- События продвижения очереди создаются для детей, чья позиция изменилась.

## Зависимости

Задачи 002, 004 и 005.

## Implementation Notes

Пересчёт должен быть атомарным: пользователь не должен увидеть промежуточное неконсистентное состояние очереди.
