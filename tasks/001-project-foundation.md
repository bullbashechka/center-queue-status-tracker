# 001. Настройка проекта и базовой архитектуры

## Этап

Фундамент продукта.

## Цель

Подготовить рабочее веб-приложение с базовой структурой, конфигурацией, окружениями и точками расширения для админки, публичных страниц, фоновых задач и интеграций.

## Контекст из PRD

Продукт состоит из внутренней админки для сотрудников и публичных страниц для родителей. Интерфейс и сообщения только на русском языке.

## Scope

- Создать базовую структуру приложения.
- Настроить конфигурацию окружения.
- Подготовить маршруты для админки и публичной части.
- Заложить место для фоновой задачи ежедневной агрегации сообщений о продвижении очереди.
- Настроить базовую обработку ошибок и страницы 404/500.

## Out of Scope

- Реализация бизнес-логики очереди.
- Реальная интеграция с внешними каналами уведомлений.
- Детальная визуальная полировка.

## Требования

- Админские маршруты должны быть отделены от публичных.
- Публичные страницы должны открываться без авторизации.
- Все пользовательские тексты должны быть на русском языке.
- Конфигурация секретов не должна храниться в коде.

## Acceptance Criteria

- Приложение запускается локально без ошибок.
- Есть базовые маршруты для админки, персональной страницы статуса и поиска по ИИН.
- Ошибка неизвестного маршрута показывает понятное русскоязычное сообщение.
- Переменные окружения задокументированы в примере конфигурации.

## Зависимости

Нет.

## Implementation Notes

Не привязывать публичные страницы к сессии сотрудника. Это разные пользовательские поверхности.
