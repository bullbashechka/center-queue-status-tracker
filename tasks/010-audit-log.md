# 010. Аудит действий сотрудников

## Этап

Заглушка уведомлений и аудит.

## Цель

Зафиксировать ключевые действия сотрудников для разбора ошибок и спорных ситуаций.

## Контекст из PRD

Журнал фиксирует добавление ребёнка, редактирование данных, удаление из активной очереди, смену статуса, откат статуса и копирование/ручную подготовку сообщения для родителя.

## Scope

- Модель записи аудита.
- Запись события при каждом ключевом действии.
- Сохранение сотрудника, даты/времени, типа действия и идентификатора ребёнка.
- Отображение времени по `Asia/Almaty`.
- Минимальный просмотр журнала сотрудником, если это нужно для проверки MVP.

## Out of Scope

- Расширенная аналитика.
- Экспорт аудита.
- Разные уровни доступа к журналу.

## Требования

- Аудит не должен блокировать основное действие при некритичной ошибке отображения.
- Время должно быть единообразным и понятным для центра.
- Журнал не должен быть публично доступен.

## Acceptance Criteria

- Добавление ребёнка создаёт запись аудита.
- Редактирование данных создаёт запись аудита.
- Удаление/архивирование создаёт запись аудита.
- Смена и откат статуса создают записи аудита.
- Копирование или ручная подготовка сообщения для родителя создаёт запись аудита.
- Запись аудита содержит сотрудника, время, тип действия и идентификатор ребёнка.

## Зависимости

Задачи 003, 005, 006 и 009.

## Implementation Notes

Не хранить в аудит-логе лишние персональные данные, если достаточно ссылки на идентификатор ребёнка.
