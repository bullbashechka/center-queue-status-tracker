# 004. Список детей, поиск и фильтры в админке

## Статус

Выполнено.

## Этап

Админка и управление очередью.

## Цель

Сделать главный рабочий экран сотрудника со списком активных детей, текущим статусом, номером в очереди, поиском и фильтрами.

## Контекст из PRD

После входа сотрудник попадает на главный экран со списком детей в очереди. В списке есть поиск по ФИО, ИИН и телефону, а также фильтр по статусу.

## Scope

- Вывести список активных детей.
- Показать ФИО, ИИН, телефон, статус, номер в очереди и количество семей впереди.
- Добавить поиск по ФИО, ИИН и телефону.
- Добавить фильтр по статусу.
- Добавить переход к действиям: добавить, редактировать, сменить статус, удалить, повторно отправить ссылку.

## Out of Scope

- Архивный список удалённых записей, если он не нужен для аудита MVP.
- Ручное изменение порядка очереди.

## Требования

- Список показывает только активные записи.
- Зачисленные дети не должны отображаться как участники активной очереди.
- Поиск и фильтры не должны раскрывать публичным пользователям какие-либо данные.

## Acceptance Criteria

- Сотрудник видит список активных детей после входа.
- Поиск находит ребёнка по части ФИО, ИИН или телефона.
- Фильтр по каждому статусу показывает только подходящие записи.
- Номер очереди и количество семей впереди совпадают с FIFO-расчётом.
- Пустой результат поиска показывает понятное сообщение.

## Зависимости

Задачи 002 и 003.

## Implementation Notes

Не добавлять ручную сортировку очереди, которая меняет FIFO-порядок.
