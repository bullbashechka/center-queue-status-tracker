# 007. Персональная публичная страница статуса

## Этап

Публичные страницы для родителей.

## Цель

Сделать страницу статуса ребёнка по уникальной ссылке без авторизации.

## Контекст из PRD

Родитель открывает персональную ссылку, полученную от сотрудника, и видит ФИО ребёнка, этапы, текущий номер в очереди, количество семей впереди и ориентировочный срок, если он указан.

## Scope

- Публичный маршрут персональной страницы по уникальному токену.
- Отображение ФИО ребёнка.
- Отображение этапов с отметкой пройденных и текущего.
- Отображение номера в очереди и количества семей впереди для активного ребёнка.
- Отображение ориентировочного срока, если он указан.
- Корректное состояние для зачисленного и удалённого ребёнка.

## Out of Scope

- Вход родителя.
- Отображение данных других детей.
- Редактирование данных родителем.

## Требования

- Страница не требует пароля.
- Для удалённой записи статус не показывается.
- Для зачисленного ребёнка страница показывает финальный статус, но не показывает номер активной очереди.
- Данные должны быть актуальны при каждом открытии страницы.

## Acceptance Criteria

- Валидная ссылка активного ребёнка открывает страницу статуса.
- Страница показывает только данные конкретного ребёнка.
- Если срок не указан, блок срока не отображается.
- Если ребёнок зачислен, номер очереди и «семей впереди» не показываются.
- Невалидная или удалённая ссылка показывает безопасное сообщение без персональных данных.
- Страница удобно читается на мобильном экране.

## Зависимости

Задачи 002 и 006.

## Implementation Notes

Не показывать ФИО или данные других семей. Только обезличенный счётчик «перед вами N семей».
